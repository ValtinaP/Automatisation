# Telegram News Parser (MVP)

Скрипт подключается к твоему личному аккаунту Telegram (не боту — это
технически необходимо, чтобы читать папки и историю каналов/групп),
слушает новые сообщения в выбранных каналах/группах/папках, прогоняет
каждое через AI (OpenRouter) на соответствие твоей инструкции и
пересылает совпадения в приватный канал.

## Установка (Windows 10)

1. Установи Python 3.11+ с python.org (отметь "Add to PATH" при установке).
2. Открой PowerShell в папке проекта и выполни:
   ```
   python -m venv .venv
   .venv\Scripts\activate
   pip install -r requirements.txt
   ```

## Настройка

1. **Telegram API-ключи**: зайди на https://my.telegram.org -> "API development
   tools", создай приложение, получи `api_id` и `api_hash`.
2. **Приватный канал для уведомлений**: создай в Telegram приватный канал
   (например, "Мои совпадения"), добавь туда себя как участника/админа.
   Узнай его `@username` (если задал) или numeric id.
3. **OpenRouter**: ключ уже есть — понадобится модель, укажи любую
   доступную (по умолчанию `openai/gpt-4o-mini`, дешёвая и быстрая).
4. Скопируй `.env.example` в `.env` и заполни все поля.
5. Скопируй `config.example.yaml` в `config.yaml`, впиши свою инструкцию
   (что тебя интересует), список каналов и/или названия папок Telegram.

## Запуск

```
python main.py
```

При первом запуске Telethon спросит номер телефона, код из Telegram и,
если включена, 2FA-пароль. После этого создаётся файл сессии
(`*.session`) — при следующих запусках повторный вход не требуется.

## Как это работает и почему не должно привести к ограничениям аккаунта

- Скрипт **не рассылает** ничего в мониторимые каналы/группы, не
  вступает/выходит из них программно, не ставит реакции — единственное
  исходящее действие — отправка себе в приватный канал.
- Новые сообщения обрабатываются через push-события Telegram
  (`NewMessage`), а не через периодический опрос истории — это то же
  поведение, что у обычного открытого клиента Telegram.
- Есть троттлинг: между AI-запросами выдерживается пауза
  (`ai_rate_limit_seconds` в `config.yaml`), сообщения обрабатываются
  по очереди, а не пачками.
- При ошибке `FloodWaitError` от Telegram скрипт ждёт ровно столько,
  сколько попросил сервер, и продолжает работу.

## Дальше по плану

Это MVP без графического интерфейса — управление через `config.yaml`.
Следующие шаги (после проверки, что ядро стабильно работает):
- десктопный GUI (PySide6): выбор каналов/папок из списка, переключение
  режима "инструкция" / "поисковый запрос", календарь дат;
- генерация отчёта за диапазон дат в `.docx` / `.pdf`;
- AI-справка по конкретному сообщению по запросу из интерфейса.
